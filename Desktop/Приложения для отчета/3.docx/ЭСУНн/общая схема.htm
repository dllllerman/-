1.1.         Общая схема работы с файлами

Работа с файлами осуществляется в следующем порядке:

1.       Определить переменные файлового типа (логические файлы).

2.       Каждому из используемых физических файлов при помощи процедуры Assign указать переменную файлового типа.

3.       Инициировать файл – значит указать направление передачи данных. Работа с файлом возможна, если он существует. Такой файл открывают, делая его доступным для ввода и/или вывода. В случае отсутствия файл должен быть создан. При открытии и создании файла для временного хранения его данных автоматически выделяется область в оперативной памяти, которая называется буфером файла.

4.       Принцип обработки файлов состоит в следующем. Данные из файла сначала считываются в оперативную память компьютера, для чего в программе назначаются переменные подходящих типов. Вся дальнейшая обработка ведется для этих переменных. В случае необходимости ее результаты записываются в новый или дописываются к уже существующему файлу.

5.       Закрытие файловой переменной и сохранение данных на диске по окончании работы с файлом. Если файл закрыт, над ним можно выполнять не все предусмотренные действия.

Для работы с файлами определен перечень общих процедур и функций, используемых независимо от типа файла. В таблице 8.1 приведен перечень вспомогательных процедур и функций для работы с файлами.

Процедура Assign (ФайловаяПеременная, ИмяФайла). Предшествует другим процедурам, т.к. ставит в соответствие физическому файлу на внешнем устройстве логический файл – файловую переменную, к которой впоследствии будут обращаться все другие файловые процедуры.

Процедура Reset (ФайловаяПеременная). Открывает существующий файл на чтение и ставит указатель на начало первого элемента файла. При отсутствии внешнего файла с указанным именем возникает сообщение об ошибке. Если при чтении файла возникнет необходимость вернуть указатель в его начало, достаточно будет просто применить процедуру reset к этому файлу еще раз.

Функция Ioresult. Проверяет существование файла на диске.

Процедура Rewrite (ФайловаяПеременная). Создает и открывает новый файл для последующей записи данных. После ее успешного выполнения файл готов к записи в него первого элемента. Если внешний файл с указанным именем уже существует, то он удаляется, и на его месте создается новый пустой файл с тем же именем.

Процедура Close (ФайловаяПеременная). Закрывает файл после его обработки. В противном случае может произойти потеря данных. Впоследствии ФайловаяПеременная может быть связана с другим физическим файлом.

Процедура Rename (ФайловаяПеременная, ИмяФайла). Переименовывает неоткрытый внешний файл. Новое имя задается строкой ИмяФайла.

Процедура Erase (файловаяПеременная). Удаляет неоткрытый внешний файл любого типа, задаваемый параметром ФайловаяПеременная.

Логическая функция Eof(ФайловаяПеременная. Выполняет проверку, не достигнут ли конец файла при чтении из него данных.
