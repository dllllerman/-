7.         Типизированные файлы

Для хранения однородной информации, например, только числовых данных определенного типа, использование текстового файла является невыгодным. Компьютер непродуктивно тратит ресурсы на постоянное преобразование данных из символьного вида в двоичный и обратно, эти преобразования могут привести к ошибкам округления, часто требуется отслеживать признаки конца строки, доступ выполняется последовательно, а значит медленно, объем файла велик.

Типизированный файл состоит из последовательности элементов одного типа и длины. Их число и, следовательно, размер файла не ограничивается при его задании. Каждый элемент файла имеет свой номер. Первый элемент считается нулевым. В каждый момент времени программа доступен только один текущий элемент, на который установлен указатель файла. После выполнения операции чтения или записи для элемента N, указатель автоматически смещается к следующему по порядку элементу N+1, который в свою очередь, становится доступным для чтения или записи.

Обратите внимание – так как все элементы файла имеют одинаковую длину, позиция каждого элемента легко вычисляется. Поэтому указатель может быть перемещен на любой элемент файла, обеспечивая к нему прямой доступ. Фактически типизированный файл – это массив на диске с неограниченным размером и сравнительно невысокой скоростью обработки. Очень часто в качестве элементов типизированного файла выступают записи, что позволяет программисту работать с базами данных.

В отличие от текстового файла типизированный файл не делится на строки. Данные в нем изначально записаны во внутреннем двоичном представлении компьютера – в виде, в котором они хранятся в оперативной памяти. Непосредственный просмотр такого файла не позволит пользователю разобраться в его содержимом. Для этого необходимо предварительно знать формат хранения данных и уметь обрабатывать его при помощи программы TP.

Объявление типизированных файлов:

Type ИмяТипа = file of Тип;

Var ФайловаяПеременная : ИмяТипа;

или

Var ФайловаяПеременная : file of Тип;
